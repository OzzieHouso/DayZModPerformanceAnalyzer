Frequently Asked Questions

Everything you need to know about the analyzer

Accuracy & Reliability

How accurate is this analyzer?

The analyzer has approximately 85-90% accuracy for detecting performance issues:

  • 95%+ accurate for critical issues (Sleep(), file I/O in loops, RPC spam)
  • 85-90% accurate for common patterns (GetPlayers() spam, world scans)
  • 70-80% accurate for context-dependent issues (scheduler patterns)

Will it flag good code?

Yes, sometimes. The analyzer is pattern-based and cannot understand context. For example:

// This is perfectly fine (one-time initialization)
void OnInit() {
  GetGame().GetObjectsAtPosition(vector.Zero, 999999, objects, null);
}

This is why we use soft language like "Recommended:" instead of "ERROR:"

Can I trust the score?

The score is informational, not absolute. Use it as a guide:

  • 90-100: Excellent practices
  • 75-89: Good, minor improvements possible
  • 60-74: Fair, review medium issues
  • 40-59: Poor, address high-severity issues
  • 0-39: Critical issues detected

Is it based on official DayZ code?

Yes! Enhanced rules are derived from official DayZ scripts:

  • Analyzed MissionServer.c for server patterns
  • Extracted update intervals (30s for stats, 1s for gameplay)
  • Documented CallQueue category usage
  • Identified scheduler patterns (SCHEDULER_PLAYERS_PER_TICK)

What can't it detect?

The analyzer has limitations:

  • Algorithmic complexity (O(n¬≤) nested loops)
  • Memory leaks from retained references
  • Issues spanning multiple files
  • Runtime-only performance problems
  • Context-specific optimizations

Using the Tool

How do I use it?

  1. Create a ZIP of your mod folder containing the scripts/ directory
  2. Upload via drag & drop or click
  3. Wait 5-30 seconds for analysis (depends on mod size)
  4. Review the results and recommendations

What should I do with the results?

‚úÖ Recommended Approach:

  1. Fix CRITICAL issues first
  2. Review HIGH severity patterns
  3. Consider MEDIUM issues if time permits
  4. Ignore LOW/INFO unless you're optimizing

Is my code uploaded to a server?

No. Analysis happens server-side but files are:

  • Deleted immediately after analysis
  • Never stored or logged
  • Not shared with third parties
  • Processed in temporary memory only

What file size limit?

Maximum 50MB per upload. Most mods are under 10MB.

If your mod exceeds this limit:

  • Upload only the scripts/ folder as a ZIP
  • Use the CLI version for local analysis (no size limit)
  • See the "Technical Details" section below for CI/CD integration

Interpreting Results

What if I disagree with a suggestion?

That's OK! The analyzer doesn't know your specific context. You may ignore suggestions if:

  • The code only runs once (initialization)
  • Your server has low player counts
  • You've tested and it performs fine
  • The pattern is intentional for your use case

What does "Recommended:" mean?

It means we suggest reviewing the code, not that it's definitely wrong. Think of it like a spell checker:

My mod got 60/100. Is it bad?

Not necessarily! The score is relative:

  • 90+ = Professional-grade practices
  • 75-89 = Good, production-ready
  • 60-74 = Fair, review suggestions
  • 40-59 = Address high-priority issues
  • <40 = Critical issues present

Focus on fixing critical issues rather than chasing a perfect score.
